# 바닥 포스터 fiducial(ArUco) 인식 — 설계 스펙 (2026-07-22, 사용자 승인)

## 배경 (근본 원인 — systematic-debugging 결과)

대형(A1급) 바닥 포스터에서 인식 불량·캐릭터 기울어짐 재발. 헤드리스 재현(변수 분리 y4m 4종)으로 확정:

| 조건 | 결과 |
|---|---|
| 정면·중형 / 정면·대형 | 정상 (3.1~3.2s, 화자 정합) — **크기는 무관** |
| **사선 45°** (이음새 유무 무관) | **교차 오인식** (라옹 포스터 → 라오나), 캐릭터 거대·기울어짐 |

원인: 카드 3종이 같은 템플릿(NFT 특징 공간에서 유사) + 사선 원근 압축 → 잘못된 타겟에 매칭된
호모그래피가 카드 평면을 틀어지게 추정. 바닥 포스터는 서서 보면 필연적으로 사선 → 구조적 실패.
기립 로직(marker.js `uprightness`)은 이미 존재하나 왜곡된 평면 위에서는 무력.

## 결정 (하이브리드)

- **손 카드 = 기존 MindAR(NFT) 유지** — 검증된 UX, 재인쇄 불필요.
- **바닥 포스터 = ArUco fiducial 4점** — 모서리마다 마커, ID가 캐릭터+모서리 인코딩
  (라옹 10~13 · 라오니 20~23 · 라오나 30~33; corner = id%10 → 0:좌상 1:우상 2:좌하 3:우하).
- 라이브러리: **js-aruco2** (MIT, 순수 JS 424KB, `--ignore-scripts` 설치 가능),
  사전 `ARUCO_MIP_36h12`.

## 성공 기준 (사용자 강조: "딱 붙어있게")

1. 라옹 포스터 → 라옹 소환 (사선 45°·이음새 조건 포함, 오인식 0)
2. 캐릭터가 포스터 중앙에 **수직 기립·정위치 고정** — 떨림·기울어짐·표류 없음
3. 일시적 검출 누락(가림 등)에도 화면에서 튀지 않음 — 마지막 포즈 유지 후 재스냅
4. 손 카드 기존 동작 무회귀 (E2E S4·S8 그린 유지)

## 구성

1. **`src/scenes/poster-detect.js`** — js-aruco2 래퍼: `<video>`를 640px 캔버스로 다운샘플 →
   `detect()` → `[{id, corners}]` → `{key, corner}` 매핑. 순수 함수부(ID 매핑·모서리 오프셋)는
   유닛 테스트 대상.
2. **`src/scenes/poster.js`** — 포스터 씬: 마커별 POSIT 6DoF(마커 실측 한 변 = 포스터 폭 14%)
   → 다중 마커 평균 + 지수 스무딩 → 포스터 중앙(코너 오프셋 역산)에 캐릭터 기립
   (업 = 포스터 평면 법선, 카메라 방향 요 빌보드). 캐릭터 키 = 포스터 폭 비례(config).
   검출 누락 N프레임까지 마지막 포즈 유지, 초과 시에도 화면 유지(기존 hold UX 답습) 후 재스냅.
3. **entry.js 배선** — [카드로 소환하기] 진입 불변. MindAR가 켠 video를 150ms 간격 샘플링,
   **3회 연속 검출 → 포스터 모드 확정**(MindAR stop, 스트림 재사용). NFT confirm 지연(1.6s) 중
   ArUco 검출 시 NFT 소환 취소(veto) — 포스터를 NFT가 잘못 무는 현행 버그 차단.
4. **인쇄물** — floor-card SVG 3종 모서리에 마커 삽입(사전 비트 매트릭스를 SVG rect로 생성,
   기존 점선 삼각 장식 대체), 분할 포스터 PDF 재생성. 손 카드 인쇄물 무변경.
5. **검증** — 유닛: ID·오프셋 매핑, 마커 비트 매트릭스 스냅샷, veto 로직.
   E2E 신규: 마커 포스터 사선 45° y4m → 포스터 모드 진입·화자 정합·기립(캐릭터 월드업 단언),
   음성통제: 마커 없는 포스터(v2/v3 y4m)는 포스터 모드 미진입. 기존 S4·S8 회귀 유지.
