# ADR 0002: 바닥 대형 포스터는 NFT 대신 ArUco fiducial로 인식한다

- 날짜: 2026-07-22
- 상태: 채택 (구현·E2E 검증 완료)

## 맥락

바닥용 대형 포스터(A4 카드의 A3+ 확대 인쇄)에서 실기기 인식 불량·캐릭터 기울어짐이 보고됐다.
헤드리스 재현(변수 분리 y4m: 정면/대형/사선/이음새)으로 원인을 확정했다:

- 크기·이음새는 무관. **사선(서서 내려다보는 45°+) 시점이 원인** — NFT(Natural Feature Tracking, 자연 특징점 추적: 그림 자체의 특징점을 지문처럼 등록해 인식) 방식에서 세 카드가 같은 디자인
  템플릿이라 NFT 특징 공간에서 유사하고, 원근 압축된 특징이 다른 카드 타겟에 교차 매칭된다
  (라옹 포스터 → 라오나 소환). 잘못된 타겟에 맞춘 호모그래피가 카드 평면을 틀어지게 추정해
  기립 로직(marker.js uprightness)까지 무력화된다.

## 결정

1. **하이브리드**: 손 카드는 검증된 MindAR(NFT) 유지, 바닥 포스터만 fiducial.
2. 포스터 모서리에 **ArUco 마커 4점**(js-aruco2, ARUCO_MIP_36h12). ID가 캐릭터+모서리를
   인코딩(십의 자리=캐릭터, 일의 자리=모서리)해 교차 오인식이 구조적으로 불가능.
3. 진입 UI 불변: [카드로 소환하기] 안에서 MindAR의 video를 150ms 샘플링, 3연속 검출 시
   포스터 모드 전환. NFT confirm은 최근 600ms 내 ArUco 검출 시 양보(veto).
4. 포즈는 마커별 POSIT가 아니라 **보이는 모든 마커 꼭짓점(4~16점)의 호모그래피 분해 1회**.
   - 근거: 평면 POSIT는 작은 정사각 하나에서 2중 해 모호성이 있음(계측: 같은 평면의 두 마커가
     서로 다른 법선 반환). 점이 포스터 전체에 퍼지면 해가 유일·안정.
   - js-aruco2 svd.js는 1-기반 인덱스 잔재로 9×9에서 범위 초과 — 야코비 고유분해 자체 구현.

## 결과

- 계측(사선+이음새 y4m): 진입 2.1s, 화자 정합(라옹), 기립 법선 upDot 0.968·표본 폭 0.004.
- E2E S12(성공 경로)·S13(마커 없는 포스터 음성통제) 추가, 전체 14종 그린.
- 부수 수정: e2e 하네스 `URL.pathname` → `fileURLToPath` (비ASCII 경로에서 %인코딩 깨짐).

## 대안 (기각)

- MindAR 타겟 보강(포스터 전용 .mind 추가): 사선 교차 매칭의 근본 원인을 못 고침.
- 전면 fiducial 교체: 검증된 손 카드 UX 재작업 + 전 카드 재인쇄 비용, 이득 없음.
